# 🚀 ResumeForge AI — JD-to-Resume Generator (SaaS Edition)

An AI-powered, multi-tenant SaaS application that generates highly tailored, ATS-optimized resumes from Job Descriptions using Retrieval-Augmented Generation (RAG).

## 🏗️ Architecture

```
User Login → Knowledge Base Ingestion → FAISS Vector DB → JD Input → Gap Analysis → LLM Generation → Tailored Resume
```

**Tech Stack:**
- **Frontend**: Streamlit
- **Database**: SQLite (User Auth, KB tracking, Resume history)
- **Vector DB**: FAISS (Isolated per user)
- **LLM**: OpenAI (GPT-4o, GPT-4o-mini)
- **Embeddings**: OpenAI (text-embedding-3-small/large)
- **Export**: python-docx (DOCX) + fpdf2 (PDF)

## 🌟 Key Features

### 1. Multi-Tenant SaaS Design
- **Authentication**: Secure login/signup system with role-based access (Admin vs. User).
- **Data Isolation**: Each user has their own private SQLite entries and separate FAISS vector stores.
- **Admin Role**: Automated loading of pre-defined root `knowledge_base/` files and system-wide analytics.

### 2. Intelligent Resume Generation
- **Target Job Role**: Guided generation based on a specific job role.
- **Gap Analysis**: Identifies missing skills and provides learning recommendations.
- **Skill Injection**: Option to "Inject Missing Skills" for realistic resume enhancement.
- **ATS Scoring**: Real-time scoring of your resume against the JD.

### 3. Professional Formatting & Export
- **Clean Headers**: Modern inline contact formatting (e.g., `Name | Email | LinkedIn | ...`).
- **Clickable Links**: All URLs (LinkedIn, GitHub, Projects) are clickable in exported PDF and DOCX files.
- **No Placeholders**: Strict generation logic ensures no `[University Name]` or `[Phone Number]` artifacts.

### 4. Admin Dashboard
- **User Management**: Overview of all registered users and their activity.
- **Resume History**: Access and download every resume generated by any user.
- **KB Sync**: Automatic bi-directional sync between Admin UI and root filesystem.

## 🚀 Quick Start

### 1. Install Dependencies
```bash
pip install -r requirements.txt
```

### 2. Set Up API Key
Create a `.env` file or update `backend/config.py` with your OpenAI key.

### 3. Run the App
```bash
python -m streamlit run app.py
```

### 4. Admin Setup
The first user with the `ADMIN_EMAIL` specified in `config.py` will automatically receive administrator privileges.

## ⚙️ Configuration

| Parameter | Description |
|-----------|-------------|
| **Generation Model** | GPT-4o for high-quality resume drafting |
| **Embedding Model** | text-embedding-3-large for precise RAG retrieval |
| **User Limit** | Configurable resume generation limits for free users |
| **Sync Logic** | Admin KB entries are automatically mirrored to the `knowledge_base/` root folder |
